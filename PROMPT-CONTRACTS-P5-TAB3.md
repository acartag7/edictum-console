## Files to Create (4 files)

### 1. `pages/contracts/diff-summary.tsx`

**Props:** `diff: ContractDiff`

Renders the contract-level change summary. This is the PRIMARY diff view — most users never need the YAML diff.

**Layout:**
```
+1 added   -0 removed   ~2 modified   5 unchanged

Added:    exec-sandbox (sandbox)
Modified: block-sensitive-reads — added .pem, id_rsa to contains_any list
Modified: session-limits — max_tool_calls 30 → 50
```

**Summary bar:** Flex row with counts. Colors:
- Added: `text-emerald-600 dark:text-emerald-400`
- Removed: `text-red-600 dark:text-red-400`
- Modified: `text-amber-600 dark:text-amber-400`
- Unchanged: `text-muted-foreground`

**Detail list:**

For added contracts:
- Show contract ID + type badge

For removed contracts:
- Show contract ID + type badge (with strikethrough or dimmed)

For modified contracts:
- Show contract ID + human-readable list of changes
- Changes come from `diff.modified[].changes[]` (generated by `diffContracts` in yaml-parser)

Unchanged: count only, no detail. Collapsible if user wants to see the IDs.

**File size target:** ~80-100 lines

### 2. `pages/contracts/diff-yaml.tsx`

**Props:** `oldYaml: string`, `newYaml: string`

Text diff view — secondary to the contract-level summary. Collapsed by default.

**Layout:**
```
▶ YAML Diff  [Side-by-side │ Unified]
```

**Implementation:**
- Outer: `Collapsible` — collapsed by default
- Toggle: small `Tabs` (two triggers: "Side-by-side", "Unified")
- Diff computation: `import { diffLines } from "diff"` (npm package, already installed)
- Render: `ScrollArea` + `<pre>` with line numbers

**Line coloring (light/dark safe):**
- Added lines: `bg-emerald-500/15` (no text color change — the bg is enough)
- Removed lines: `bg-red-500/15`
- Context lines: no background

**Side-by-side mode:**
- Two columns: old (left), new (right)
- Synchronized scroll if possible (or just two independent `ScrollArea`s)
- Added lines highlighted in right column, removed in left

**Unified mode:**
- Single column, interleaved
- `-` prefix for removed, `+` prefix for added

**Line numbers:** `text-muted-foreground text-xs w-8 text-right inline-block mr-2`

**File size target:** ~100-130 lines

### 3. `pages/contracts/diff-impact.tsx`

**Props:** `oldYaml: string`, `newYaml: string`, `bundles: BundleWithDeployments[]`

Impact preview: evaluate recent events against both versions, show which verdicts would change.

**This is the most complex component.** It makes N API calls, needs progress tracking, and must handle partial failures gracefully.

**Flow:**
1. User clicks "Run Impact Analysis" button
2. Fetch recent events: `listEvents({ limit: 50 })`
3. For each event, call `evaluateBundle` twice (old YAML, new YAML)
4. Compare verdicts
5. Display results

**State machine:**
```typescript
type ImpactState =
  | { status: "idle" }
  | { status: "running"; progress: number; total: number }
  | { status: "done"; results: ImpactResults }
  | { status: "error"; message: string }
```

**Progress:** Show `Progress` component (shadcn) or a simple bar: "Evaluating 23/50..."

**Partial failure handling:**
- Track per-event eval errors in a separate array
- If an event fails (e.g., one version's YAML can't evaluate the tool call), increment error count but continue
- Results display: "42/50 evaluated successfully, 8 failed"
- Collapsible section showing which events failed and why

**Results display:**
```
Based on 50 most recent events:
42/50 evaluated, 8 failed (parse error)

12 events would change verdict
• 8: allowed → denied (block-sensitive-reads)
• 4: denied → allowed (session-limits relaxed)

[View affected events →]
```

- Summary: N unchanged, N new denials, N relaxed (newly allowed), N failed
- Changed verdicts: grouped by direction (allowed→denied, denied→allowed), with contract attribution
- "View affected events →" → link to Events feed (future enhancement, placeholder for now)

**Evaluate endpoint unavailable:**
- If `evaluateBundle` returns 404 on first call, show placeholder: "Impact analysis requires the evaluate endpoint. Deploy the backend first."
- Don't make the remaining N calls

**File size target:** ~130-170 lines (this is the densest component). If over 180, extract the results display to `diff-impact-results.tsx`.

### 4. `pages/contracts/diff-tab.tsx`

**Props:** `bundles: BundleWithDeployments[]`

Orchestrator for the three diff sections.

**Layout:**
```
Compare: [v3 ▾] → [v5 ▾]    [Swap ⇄]

[DiffSummary]
[DiffImpact]
[DiffYaml] (collapsed)
```

**Version selectors:** Two shadcn `Select` components, pre-populated with all bundle versions.

**Pre-population from URL:**
```typescript
const [searchParams] = useSearchParams()
const fromParam = searchParams.get("from")
const toParam = searchParams.get("to")

const [fromVersion, setFromVersion] = useState<number | null>(fromParam ? Number(fromParam) : null)
const [toVersion, setToVersion] = useState<number | null>(toParam ? Number(toParam) : null)
```

Default if no URL params: from = second-latest version, to = latest version. If only one version exists, show message "Need at least two versions to compare."

**Swap button:** `Button variant="ghost" size="icon"` with `ArrowLeftRight` icon → swaps from/to.

**Data loading:**
```typescript
useEffect(() => {
  if (!fromVersion || !toVersion || fromVersion === toVersion) return
  Promise.all([
    getBundleYaml(fromVersion),
    getBundleYaml(toVersion),
  ]).then(([oldYaml, newYaml]) => {
    setOldYaml(oldYaml)
    setNewYaml(newYaml)
    const oldBundle = parseContractBundle(oldYaml)
    const newBundle = parseContractBundle(newYaml)
    setDiff(diffContracts(oldBundle, newBundle))
  }).catch(e => setError(e.message))
}, [fromVersion, toVersion])
```

**Conditional rendering:**
- Same version selected: "Select two different versions to compare."
- Loading: `Loader2` per section
- Error: error message with retry

**File size target:** ~90-120 lines
